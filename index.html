Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name : creates a new branch and switches it to
  • git add -A : commit all files in the working branch
  • git commit -m : use to associate a message with our commit
  • -m is actually just to associate a "message"
  • using class attributes allows us to only add changes to a certain elements but not all the elements
  • How to make a new branch in the command line
  • change directories to where you want to be (cd directoryx ) type (git status) to check which branch you are currently in, you should be on the main branch if you are not switch to the main branch by entering git checkout main in the command line pull down the latest main brain by entering git pull origin main from the main branch, use git checkout -b branchyouwant (ex: to add a branch named feature/add-html git checkout -b featrue/add-html) confirm you are on the new branch by typing git status
  • after all of this use git add -A to commit files in the working branch then git commit our changes to our repository use the command git pull origin main to pull the base branch you should see something along the lines of already up to date to push up all of our changes by using the command git push origin branchyouwant
